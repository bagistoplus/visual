Custom Product, Category, and Page Templates

Current design for Shopify-style custom templates in Visual: template content stays file-based, assignment uses Bagisto-native model storage, and creation happens from a nested template selector in the Visual Editor.

Types: product, category, page Theme files stay file-based No custom index Assignment UI: entity forms Editor override: _template

Locked Scope

Custom template support is limited to assignable entity page types. Homepage/index remains a core editor template only and is not part of custom template creation or assignment.

  • Supported custom types: product, category, page.
  • Unsupported for this phase: custom index, delete, rename, assignment counts.
  • Custom templates replace only the inner Visual content include.
  • Template assignment changes publish immediately with Bagisto entity forms.

Storage Decision

Use Bagisto-native storage so fetched entities already carry their assignment data where possible. Avoid Concord model overrides by default.
  • Product: product attribute visual_template, channel + locale aware.
  • Category: category_translations.visual_template, locale aware.
  • CMS page: cms_page_translations.visual_template, locale aware.
  • Assignments store the full key, e.g. gift-box.product; default stores null.

Storefront Data Flow

The page wrapper should not resolve assignment through a late Blade query. The assignment value should come from the same Bagisto data the model already uses.

1. Bagisto Fetches Entity Product attributes or translations are loaded by existing model behavior.
→
2. Validate Template Key Strict type match and live template availability decide whether to use the assignment.
→
3. Wrapper Includes Template Blade includes the assigned template key or falls back to product, category, or page.

Template Loading Model

Templates are loaded through one registry service that discovers definitions from package files and Visual storage. It exposes type-filtered options to the editor and Bagisto entity forms.

1. Discover Files Scan theme package, live storage, or editor storage depending on request context.
→
2. Parse Template Key landing.product.json becomes key landing.product, type product, name landing.
→
3. Return Registry Registry returns default templates plus custom templates filtered by type and active channel theme.

Discovery Precedence

Discovery includes fallback channel/locale paths. Later sources override earlier ones for the same full template key.

Order Source Used When
1 Theme package defaults Always available as fallback
2 Live storage Normal storefront/admin assignment forms
3 Editor storage Visual Editor and design mode only
  • Bagisto entity forms list live templates only.
  • Visual Editor includes editor drafts.
  • Package-default custom templates are assignable.

Registry Service Sketch

final class TemplateDiscovery
{
    public function forType(
        string $themeCode,
        string $type,
        ?string $channel = null,
        ?string $locale = null,
        bool $includeEditorDrafts = false
    ): Collection {
        return $this->all($themeCode, $channel, $locale, $includeEditorDrafts)
            ->where('type', $type)
            ->values();
    }

    public function all(
        string $themeCode,
        ?string $channel = null,
        ?string $locale = null,
        bool $includeEditorDrafts = false
    ): Collection {
        $templates = collect($this->defaultTemplatesForAssignableTypes());

        foreach ($this->packageTemplateFiles($themeCode) as $file) {
            $templates->put($this->keyFromFile($file), $this->templateFromFile($file));
        }

        foreach ($this->storageTemplateFiles($themeCode, 'live', $channel, $locale) as $file) {
            $templates->put($this->keyFromFile($file), $this->templateFromFile($file));
        }

        if ($includeEditorDrafts) {
            foreach ($this->storageTemplateFiles($themeCode, 'editor', $channel, $locale) as $file) {
                $templates->put($this->keyFromFile($file), $this->templateFromFile($file));
            }
        }

        return $templates->values();
    }
}
Registry labels are title-cased from the filename custom name: gift-box.product.json becomes Gift Box. Metadata labels are ignored for this phase.

Assignment Persistence

Assignment fields are injected with Bagisto view render events and saved with Bagisto update events. Invalid assignment values are skipped with a warning rather than failing the main entity save. 

Event::listen('catalog.product.update.after', function ($product) {
    // Save request('visual_template') into the product attribute value
    // for the selected product edit channel + locale.
});

Event::listen('catalog.category.update.after', function ($category) {
    // Save request('visual_template') into category_translations
    // for request('locale').
});

Event::listen('cms.page.update.after', function ($page) {
    // Save request('visual_template') into cms_page_translations
    // for the current requested locale.
});
Form placement: product general panel; category right settings panel after Visible in menu; CMS page right general panel after channel selection.

Create Template Modal

Template creation is launched from the nested selector, not from the current preview page. After creation, the editor switches to the new template.

  • User enters a name; slug is generated and editable before create.
  • Name length max is 25 characters.
  • New templates are written as {slug}.{type}.json.
  • No delete and no rename UI in this phase.

Create/Duplicate Data

  • Shared regions and shared blocks are copied from the current page data.
  • Duplicate copies non-shared regions from the selected Based on template.
  • Empty creates shared regions plus empty non-shared main.
  • Multiple non-shared regions are supported when present in a base template.
  • Create/duplicate writes only to the current editor channel + locale.

Preview Override

The editor uses _template as the source of truth for previewing unassigned custom templates. It is honored only in design/editor mode and must match the current route type. 

/red-shirt?_designMode=my-theme&_template=gift-box.product
  • Design mode priority: valid _template, then entity assignment, then default type.
  • Storefront priority: entity assignment, then default type.
  • Injected page data reports template.name = gift-box.product.
  • Persistence writes to templates/gift-box.product.json.

Strict Matching

  • Product form lists and accepts only *.product.
  • Category form lists and accepts only *.category.
  • CMS page form lists and accepts only *.page.
  • Missing or wrong-type assigned templates fall back to the default type.

Rejected Alternatives

Alternative Status Reason
Dedicated assignment table for all entities Rejected for now Would require explicit eager-loading or joins to avoid extra queries.
Concord model overrides Rejected by default Only one model override wins per contract, creating package conflict risk.
Custom homepage/index templates Out of scope No assignment target needed for this phase.
Template delete/rename UI Out of scope Rename/delete would require cross-entity assignment cleanup.
